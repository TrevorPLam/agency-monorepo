# e8-apps-studio: Constraints

## Hard constraints

- Keep schema ownership separate from presentation ownership.
- Keep client and environment boundaries explicit.
- Do not embed Studio by default just because it is technically possible.
- Do not assume editorial tooling is a shared platform before the first approved CMS surface exists.

## Documentation constraints

- Record preview and visual-editing rules explicitly.
- Keep token and secret handling guidance visible.
- Align tenant-sensitive content rules with `a5-docs-tenant-isolation-data-governance`.
