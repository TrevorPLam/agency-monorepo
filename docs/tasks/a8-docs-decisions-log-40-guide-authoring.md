# a8-docs-decisions-log: Guide

## When to use this task family

Use this family when you need to:

- Record a non-obvious technical choice that is not yet an ADR
- Clarify how a task family should reference a decision note
- Decide whether a decision should stay lightweight or promote to an ADR

## Entry workflow

1. Confirm the choice is more than a chat note but not yet a durable ADR.
2. Add a decision entry using the template in `docs/decisions/01-config-biome-migration-50-ref-quickstart.md`.
3. Update `docs/DECISION-STATUS.md` if the decision status changes.
4. Promote the entry to an ADR later if its blast radius or permanence grows.

## Review questions

- Is the item actually a decision, not just an open question?
- Does the status register need an update too?
- Should the decision stay lightweight, or is it already ADR-worthy?
