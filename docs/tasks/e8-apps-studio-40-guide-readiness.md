# e8-apps-studio: Guide

## When to use this task family

Use this family when the first Sanity-backed site is approved and the repo needs to decide whether to activate Studio.

## Readiness workflow

1. Confirm a real editorial workflow exists.
2. Confirm which schemas belong in `51-data-cms`.
3. Choose separate versus embedded Studio using the default rule and exception criteria.
4. Document preview, draft, revalidation, and token handling before activation.

## Review questions

- Is the editing workflow broad enough to justify a separate Studio?
- Would embedding improve operator experience without harming isolation?
- Are content ownership and environment boundaries explicit?
