# a7-docs-analytics-guides: Constraints

## Hard constraints

- Keep analytics docs above package implementation detail and below architecture-level rationale.
- Do not assume attribution, consent bridging, experimentation, or RUM are default workstreams.
- Do not collapse Plausible and PostHog into a fake one-size-fits-all provider story.
- Do not bury analytics governance inside marketing-only documentation.

## Documentation constraints

- Keep event naming simple and deterministic.
- Prefer additive guidance over large speculative taxonomy schemes.
- Use the same provider-boundary language across docs and task families.
