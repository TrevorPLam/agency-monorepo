# a8-docs-decisions-log: Constraints

## Hard constraints

- Do not duplicate accepted ADR content here.
- Do not record vague brainstorming as a decision.
- Do not let decisions bypass `docs/DECISION-STATUS.md`.
- Do not treat a log entry as implementation approval.

## Documentation constraints

- Keep entries lightweight and time-scoped.
- Require owner and review-date fields.
- Promote durable, high-blast-radius, or hard-to-reverse decisions to ADRs.
