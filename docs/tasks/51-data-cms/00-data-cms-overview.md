# Data Cms


## Purpose
Reusable Sanity schema fragments and shared content models for Sanity-backed client sites.


## Condition Block

- **Build when:** The first Sanity-backed client site arrives and content modeling begins.
- **Do not build when:** Content lives comfortably inside a single app without reuse needs.
- **Minimum consumer rule:** One Sanity-backed site is sufficient — schema reuse can start there and grow.
- **Exit criteria:**
  - [ ] Shared schema fragments defined (SEO fields, image galleries, author models)
  - [ ] Schema used by at least one live content app
  - [ ] TypeScript types generated from Sanity schemas
  - [ ] README with schema extension guide
  - [ ] Changeset documenting initial release
